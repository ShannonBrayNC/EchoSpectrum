# EchoSpectrum Architecture Standards

## Core Principles

- receive-only governance posture
- ETS-aligned trust propagation
- SignalForge-compatible orchestration
- modular service isolation
- replay-safe telemetry workflows
- deterministic synthetic testing

## Service Standards

Each service should expose:
- package exports
- DTO/schema separation
- governance metadata support
- configuration abstraction
- unit tests

## Frontend Standards

- React + TypeScript
- routing-aware structure
- replay-safe state handling
- governance overlays visible
- observatory-first UX

## Governance Standards

All services must:
- preserve audit metadata
- maintain RX-only posture
- expose policy identifiers
- support future trust-chain propagation

## Deployment Standards

- environment-specific configuration
- Azure-ready deployment patterns
- SignalForge orchestration compatibility
- telemetry retention configuration
